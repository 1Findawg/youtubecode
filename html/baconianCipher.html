1Findawg - Baconian Cipher

Introduction:

For the Baconian Cipher the plaintext is turned into five letter representations of each letter. So the size of the message will be multiplied by 5. Then a secondary sentence is needed. Following the original cipher an "a" will make the same index that correlates to the secondary sentence a lowercase and a "b" will make it an uppercase.

THE -> baabaaabbbaabaa (Secondary Sentence: Findawg coded me) -> FinDawg CODed Me

To decode we notice that some data is lost. The ciphertext is turned back in "a's" and "b's". Since the spaces from the plaintext are not persisted to the ciphertext they will not be in the plaintext when we decode. (Spaces in the ciphertext come from the secondary sentence) Then the "a" and "b" string is padded with "a's" to be a multiple of 5. Lastly, the "a" and "b" string is split on every 5th letter to map back to plaintext letters.
