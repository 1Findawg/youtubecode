1Findawg - Rail-fence Cipher

Introduction:

The Rail-fence Cipher does not re-map the plaintext letters to new ciphertext letters. This cipher is actually re-organizing the letters in a specifc pattern based on the number of rails (rows). A padding needs to be added to the end of the plaintext. This means adding space characters to the plaintext until the length is a multiple of (2 * (number of rails -1)). A zig-zag pattern is created as tall as the number of rails selected. Then the letters of the top-most rail are first in the ciphertext with the rail under that rail added next in the ciphertext. This is repeated until all rails are added to the ciphertext.

To decode the ciphertext make sure that the length is a multiple of (2 * (number of rails -1)). Then find K by calculating the length of the ciphertext) / (2 * (number of rails -1)). The first and last rails wil have K letters. All middle rows will have 2K letters. Then just expand the spacing per rail so that you can read the plaintext in the zig-zag pattern.
